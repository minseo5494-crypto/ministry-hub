# 기능 계획 — 클릭(가이드) 트랙

**작성**: 2026-07-21 (5차 세션)
**상태**: 설계 (구현 착수 전 범위·데이터 확정용)

---

## 1. 목표

팀이 다 같이 들으며 합주/리허설할 수 있는 **가이드 트랙**을 곡별로 생성한다.
- **클릭(메트로놈)**: BPM·박자표에 맞춘 클릭음(강박/약박 구분)
- **섹션 가이드**: 각 섹션 진입 직전 "벌스", "코러스", "브릿지" 등 음성 큐
- **재생/내보내기**: 앱에서 함께 재생 + 오디오 파일(WAV) 다운로드로 공유

---

## 2. 기술 접근 — Web Audio API (외부 라이브러리 불필요)

```
[입력] BPM · 박자표 · 섹션 구조(섹션별 마디 수) · 카운트인 마디 수
   → Web Audio 스케줄러
       ├─ 클릭: Oscillator/버퍼로 비트마다 클릭(1박 강조)
       └─ 섹션 큐: 섹션 시작 1마디 전에 음성 삽입
   → [재생] 실시간 AudioContext
   → [내보내기] OfflineAudioContext 렌더 → WAV 인코딩 → 다운로드
```

- **클릭 합성**: 정확한 타이밍은 `AudioContext.currentTime` 기반 look-ahead 스케줄러(표준 패턴).
- **섹션 음성 큐 옵션**:
  - (A) **브라우저 TTS**(`SpeechSynthesis`) — 구현 간단, 단 **OfflineAudioContext로 캡처 불가** → 실시간 재생엔 OK, WAV 내보내기엔 부적합.
  - (B) **사전 녹음 샘플**("벌스/코러스/브릿지/인트로/아웃트로" 등 짧은 오디오) — 버킷에 호스팅, 버퍼로 삽입 → **재생·내보내기 모두 가능**(권장, 내보내기 필요 시).
  - (C) 서버 TTS 생성 후 삽입 — 유연하나 인프라 추가.
- **박자 강세**: 4/4면 1박 강함 + 2·3·4 약함. `time_signature`에서 분자 파싱.

---

## 3. 입력 데이터

| 항목 | 출처 | 비고 |
|---|---|---|
| BPM | `songs.bpm` (number) | 없으면 사용자 입력 |
| 박자표 | `songs.time_signature` ("4/4") | 없으면 기본 4/4 |
| 섹션 구조(섹션별 마디 수) | **없음(현재)** | 핵심 미확보 데이터 — 아래 참조 |
| 카운트인 | 사용자 설정(기본 1~2마디) | |

### ⚠️ 핵심 의존성 — "섹션별 마디 수"
현재 DB에 **섹션이 몇 마디인지** 데이터가 없다(`song_structure`는 `{섹션:가사}`뿐).
확보 경로:
1. **[[코드악보_자동변환_계획]] 결과 재사용** — ChordChart의 `measures[].section`으로
   섹션별 마디 수를 그대로 산출(가장 깔끔한 시너지).
2. **수동 입력 UI** — "인트로 4마디 / 벌스 8마디 / 코러스 8마디…" 간단 입력.
3. 송폼(`songForms: string[]`)은 **순서**만 있고 마디 수 없음 → 마디 수는 별도 필요.

→ **MVP는 수동 입력**으로 시작하고, 기능1 완성 시 자동 채움을 얹는 게 현실적.

## 4. 데이터 모델 (곡별 가이드 설정, 선택)

```ts
interface GuideTrackConfig {
  song_id: string
  bpm: number
  time_signature: string        // "4/4"
  count_in_bars: number         // 카운트인
  sections: GuideSection[]
  voice_cue: boolean            // 섹션 음성 큐 on/off
  cue_lead_beats: number        // 섹션 몇 박 전에 큐 (기본 1마디)
}
interface GuideSection {
  label: string                 // "Verse1" | "Chorus" ...
  bars: number                  // 마디 수
}
```
- 저장: 신규 테이블 `song_guide_tracks`(song_id, config jsonb, created_by/at) 또는
  곡 편집 시 즉석 생성(무저장). MVP는 **무저장 즉석 생성**으로 시작 가능.

---

## 5. UI 흐름

1. 곡 상세에 **"가이드 트랙" 버튼**
2. BPM·박자표 자동 채움(없으면 입력) + **섹션/마디 수 입력**(또는 코드악보에서 불러오기)
3. 옵션: 카운트인, 음성 큐 on/off, 클릭 음색
4. **재생**(플레이어: 현재 마디/섹션 하이라이트) / **WAV 다운로드**
5. iOS 주의: 오디오는 사용자 제스처(탭) 후 `AudioContext.resume()` 필요

---

## 6. 범위 (MVP → 확장)

**MVP**
- [ ] Web Audio 클릭 엔진(BPM·박자표·강약, look-ahead 스케줄러)
- [ ] 섹션/마디 수 **수동 입력** + 카운트인
- [ ] 섹션 음성 큐 (실시간: TTS 또는 사전샘플)
- [ ] 앱 내 재생 + 마디/섹션 진행 표시

**확장(후속)**
- [ ] WAV 내보내기(OfflineAudioContext) — 사전샘플 방식이면 자연스러움
- [ ] 코드악보에서 섹션/마디 자동 채움([[코드악보_자동변환_계획]] 시너지)
- [ ] 클릭 음색/볼륨, 셔플/8비트 세분화
- [ ] 가이드 멜로디(음원) 삽입 — 범위 큼, 신중
