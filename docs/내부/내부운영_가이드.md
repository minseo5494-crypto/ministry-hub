## 1. 저작권 신고 처리 절차

### 1.1 신고 접수

**접수 채널**
- 이메일: support@worsheep.org (또는 support@worsheep.org)
- 서비스 내 신고 기능

**필요 정보**
| 항목 | 필수 여부 | 설명 |
|------|----------|------|
| 신고자 정보 | 필수 | 이름, 연락처 |
| 저작권자 확인 | 필수 | 본인이 저작권자임을 증명 |
| 침해 콘텐츠 | 필수 | 곡 제목, URL 등 특정 |
| 원저작물 정보 | 권장 | 원곡 정보, 저작권 증빙 |

### 1.2 처리 절차

```
1. 신고 접수 (D+0)
   ↓
2. 신고 내용 검토 (D+1~2)
   - 신고자가 저작권자인지 확인
   - 침해 여부 판단
   ↓
3. 콘텐츠 조치 (D+3 이내)
   - 침해 확인 시: 즉시 비공개/삭제
   - 침해 불명확 시: 추가 확인 요청
   ↓
4. 당사자 통보 (D+3)
   - 신고자에게 처리 결과 통보
   - 업로더에게 삭제 사유 통보
   ↓
5. 이의 신청 처리 (D+14까지)
   - 업로더가 이의 신청 시 검토
   - 정당한 권리 증명 시 복구 가능
```

### 1.3 처리 기준

| 상황 | 조치 |
|------|------|
| 명백한 저작권 침해 | 즉시 삭제, 업로더에게 경고 |
| 공식 악보와 중복 | 사용자 업로드 버전 삭제, 공식으로 대체 |
| 신고자 권한 불명확 | 저작권 증빙 요청 후 처리 |
| 퍼블릭 도메인 주장 | 저작권 만료 여부 확인 후 처리 |

### 1.4 반복 침해자 제재

| 횟수 | 조치 |
|------|------|
| 1회 | 경고 + 해당 콘텐츠 삭제 |
| 2회 | 업로드 기능 30일 제한 |
| 3회 | 계정 영구 정지 |

## 2. 콘텐츠 관리

### 2.1 공식 악보 등록

**등록 절차**
1. 팀과 파트너십 체결 (동의서 수령)
2. 악보 파일 수집 (팀 제공 또는 공식 채널에서 수집)
3. 메타데이터 입력 (제목, 아티스트, Key, BPM 등)
4. 관리자 페이지에서 "공식 악보" 지정
5. 저작권자 정보 입력

**메타데이터 필수 항목**
- 곡 제목 (한글)
- 아티스트/팀명
- Key
- 저작권자 정보

**메타데이터 권장 항목**
- BPM
- 박자
- 템포 (느린/보통/빠른)
- 테마
- 시즌
- 유튜브 링크
- 가사

### 2.2 사용자 업로드 악보 관리

**승인 기준**
| 항목 | 기준 |
|------|------|
| 파일 형식 | PDF, JPG, PNG (10MB 이하) |
| 악보 품질 | 가독성 있는 상태 |
| 중복 확인 | 동일 곡 공식 악보가 있으면 거부 |
| 저작권 | 업로더 책임 (약관 동의 확인) |

**승인 절차**
1. 사용자 업로드 → `song_approval_requests` 테이블에 저장
2. 관리자 페이지에서 검토
3. 승인 시 `songs` 테이블로 이동
4. 거부 시 사유 입력 후 업로더에게 통보

### 2.3 콘텐츠 삭제 기준

| 사유 | 조치 |
|------|------|
| 저작권 침해 신고 | 즉시 삭제 (3영업일 내) |
| 품질 불량 (가독 불가) | 삭제 또는 교체 요청 |
| 중복 콘텐츠 | 품질 좋은 버전 유지, 나머지 삭제 |
| 부적절한 내용 | 즉시 삭제 |

## 4. 정산 관리

### 4.1 사역 수익 풀 (Royalty Pool)

WORSHEEP는 구독 및 광고 수익을 포함한 플랫폼 전체 수익 중 일부를
악보 콘텐츠 정산을 위한 **사역 수익 풀(Royalty Pool)**로 운영한다.

해당 수익 풀은 특정 수익 항목(구독료, 광고 수익 등)을 직접적으로 분리하여 산정하지 않으며,
플랫폼 내에서 발생한 악보 사용 행위 데이터를 기반으로 한 **정산 지수**에 따라 배분된다.

### 4.2 정산 지수 (Settlement Index)

정산 지수는 단순 조회 수가 아닌, 실제 예배 준비 및 사용과 밀접한 행위를 중심으로 산정된다.

| 행동 | 설명 |
|------|------|
| 악보 열람 | 악보 상세 페이지 조회 |
| 악보 다운로드 | PDF/이미지 다운로드 |
| 악보 필기 및 주석 | 필기 도구 사용, 메모 추가 |
| 콘티 포함 | 해당 악보가 콘티에 등록됨 |
| 예배 실행 | 예배일 기준 콘티 실행 (스트리밍) |

**좋아요(선호도) 지표:**
- 악보에 대한 이용자의 선호를 나타내는 보조 지표
- 정산 지수 산정 시 참고 또는 보정 요소로만 활용
- 단독으로 정산 금액을 결정하지 않음

### 4.3 조건부 수익 비율 상향

사역팀은 기본 정산 비율 외에도, 다음 조건 충족 시 사역 수익 풀 내에서 **최대 70%**까지 수익 비율 상향이 가능하다.

| 조건 | 설명 |
|------|------|
| 신곡/편곡 우선 공개 | WORSHEEP에서 최초 또는 우선 공개 |
| 공식 악보 제공 | 팀 직접 제공 공식 악보 |
| 홍보 협력 | 플랫폼 마케팅/홍보 활동 참여 |

※ 강제 조건 없이 자율적으로 참여 가능

### 4.4 정산 기준

- **정산 주기**: 월별 집계, 분기별 정산 (분기 종료 후 익월 15일)
- **최소 정산 금액**: 10,000원 (미달 시 이월)
- **정산 방식**: 계좌 이체

### 4.5 정산 유의사항

- 정산 지수는 **상대 평가 방식**으로 산정됨
- 플랫폼 정책, 서비스 구조, 이용 패턴 변화에 따라 정산 지수 산정 방식 조정 가능
- 조정 사항은 사전 고지되며, 이미 발생한 정산 내역에 소급 적용되지 않음

## 5. 베타 테스트 운영

### 5.1 테스터 관리

| 항목 | 내용 |
|------|------|
| 목표 인원 | 20팀, 50명 |
| 모집 방식 | 교회 네트워크, 지인 추천 |
| 테스터 혜택 | 전 기능 무료, 정식 출시 할인 |

### 5.2 피드백 수집

**수집 채널**
- 서비스 내 피드백 기능
- 이메일
- 카카오톡 단톡방

**피드백 분류**
| 분류 | 우선순위 | 대응 |
|------|----------|------|
| 버그 (서비스 불가) | 긴급 | 24시간 내 수정 |
| 버그 (일반) | 높음 | 1주일 내 수정 |
| 개선 제안 | 중간 | 검토 후 로드맵 반영 |
| 기능 요청 | 낮음 | 검토 후 판단 |

### 5.3 버전 관리

- 주요 업데이트: 버전 번호 변경 (v1.0 → v1.1)
- 마이너 수정: 버전 유지, 패치 노트 작성
- 업데이트 공지: 단톡방 + 이메일
