# 기능 계획 — 연주 모드 (송폼 기반 클릭 + 코드악보 실시간 싱크)

**작성**: 2026-07-22
**상태**: 설계 확정 (Q&A 반영), 1단계 착수

---

## 1. 목표

사용자가 짜놓은 **송폼(진행 순서)** 대로 클릭 트랙이 재생되고, 재생 중 **코드악보를 같이
보여주며 지금 연주해야 할 위치를 실시간 하이라이트**하는 통합 "연주 모드".

세 조각이 맞물린다:
- **코드악보**([[코드악보_자동변환_계획]]): 섹션별 **마디 수**(실제 곡 구조)
- **송폼**(`songForms[songId]`): 곡 진행 순서(I-V1-C-V2-C-B-C…)
- **클릭 엔진**(`useClickTrack`, [[클릭_가이드트랙_계획]]): 재생 중 현재 섹션/마디/박 실시간 제공

```
코드악보(섹션별 마디수) ┐
                        ├─→ 송폼 순서대로 클릭 섹션 생성(실제 마디수)
송폼 진행순서           ┘        │
                                 ├─→ 재생하면서
                                 └─→ 악보 표시 + "지금 치는 마디" 하이라이트(오디오와 싱크)
```

## 2. 확정된 결정 (Q&A)

| 항목 | 결정 |
|---|---|
| 섹션별 마디 수 출처 | **코드악보 있으면 자동, 없으면 수동 입력** |
| 절 반복(V1/V2 등) | 악보엔 Verse **한 번만** 그려짐 → V1·V2 둘 다 그 Verse 마디수 재사용 |
| 플레이어 UI | **새 전체화면 '연주 모드'** (큰 악보+하이라이트, 하단 클릭 컨트롤) |
| 코드악보 저장 | **지금 구축**(곡별 저장/불러오기) — 통합의 전제 |

## 3. 단계별 구현

### 1단계 — 코드악보 저장/불러오기 (전제조건)
- `song_chord_charts` 테이블 적용(`supabase/migrations/20260722_song_chord_charts.sql`, `unique(song_id)` 추가).
- 저장/불러오기는 **클라이언트 직접**(RLS가 곡 접근권으로 보호): `supabase.from('song_chord_charts')`.
- `ChordChartModal`: 열 때 저장된 차트 있으면 불러와 표시(재생성 스킵), "저장"·"다시 생성" 버튼.

### 2단계 — 송폼 ↔ 코드악보 섹션 매핑 → 클릭 마디수
- 코드악보를 섹션별로 묶어 **섹션당 마디 수** 산출(인쇄 섹션 기준). 예: {Intro:4, Verse:8, Chorus:8}.
- 송폼 약어(V1,V2,C,B,Pc…) → 코드악보 섹션명(Verse 1/Chorus…) 매칭.
  - `sectionFamily`로 계열 비교(V1·V2 → verse 계열 → 'Verse' 마디수 공유).
  - 매칭 실패 섹션은 `defaultBarsFor` 폴백.
- `GuideTrackModal`/연주모드가 클릭 섹션을 **실제 마디수**로 구성(기존 기본값 대신).

### 3단계 — 연주 모드 (싱크 플레이어)
- 새 전체화면 컴포넌트: 상단 큰 **ChordChartView**(하이라이트 지원), 하단 클릭 컨트롤(재생/정지/BPM/타임라인).
- `useClickTrack` 재사용. 재생 위치 매핑:
  - 클릭 현재 상태 = (송폼 섹션 index, 섹션 내 마디, 박)
  - → 송폼 섹션의 계열 → 코드악보 해당 섹션의 마디들 → (섹션 내 마디 offset)번째 마디 **하이라이트**
- 오디오/UI 동일 타이밍 갱신 → 싱크 자연스러움(기존 플레이헤드와 동일 원리).
- 현재 마디로 **자동 스크롤**(악보가 길면).

## 4. 핵심 매핑 로직

```
// 코드악보 → 섹션별 마디 수
sectionBars: Map<family, ChordMeasure[]>  // 인쇄 섹션 carry-forward로 그룹
// 송폼 → 클릭 섹션
clickSections = songForm.map(abbr => {
  const fam = sectionFamily(abbr)
  const measures = sectionBars.get(fam)
  return { label: abbr, bars: measures?.length ?? defaultBarsFor(abbr), family: fam }
})
// 재생 하이라이트: (clickSectionIdx, barInSection) → sectionBars.get(fam)[barInSection-1] 마디 하이라이트
```

## 5. 미결/엣지 (구현 중 처리)
- 송폼에 있는데 코드악보엔 없는 섹션(예: Rap) → 기본 마디수 + 하이라이트 없음(또는 회색).
- 코드악보 섹션이 인쇄 안 된 곡 → 하이라이트 불가(클릭만). 안내 표시.
- 한 마디 여러 코드 → 마디 단위 하이라이트(코드 단위는 후속).
- 절별 가사 전환(V1은 1절 가사, V2는 2절 가사) → MVP는 악보의 원래 가사 그대로(절 전환 표시는 후속).
