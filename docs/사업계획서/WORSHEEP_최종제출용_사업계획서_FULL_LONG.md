
# WORSHEEP 최종 제출용 사업계획서 (FULL VERSION)

## Executive Summary

WORSHEEP는 교회 찬양팀을 위한 올인원 악보 관리·협업 SaaS이자,
악보 저작권을 기반으로 한 수익 분배 플랫폼이다.

기존 CCM 시장은 음원과 영상 중심으로 저작권 체계가 발전해 왔으나,
예배 현장에서 실제로 가장 많이 사용되는 '악보'는
여전히 저작권 보호의 사각지대에 머물러 있다.

WORSHEEP는 악보 사용 데이터를 기반으로
저작권을 정산하는 새로운 기준을 제시하며,
교회, 사역팀, 창작자 모두가
윤리적·재정적으로 지속 가능한 구조를 만드는 것을 목표로 한다.

### 핵심 지표

## Ⅰ. 사업 배경 및 문제 정의

한국 교회 예배 현장에서 찬양은 예배의 핵심 요소 중 하나로 자리 잡아 왔다.
특히 1990년대 이후 CCM(Contemporary Christian Music)의 확산과 함께
워십팀 중심의 예배 문화가 정착되었으며,
현재 대부분의 중·대형 교회는 정규 찬양팀을 운영하고 있다.

### 전문 찬양팀의 부상

최근 메가처치를 중심으로 현대적이고 전문적인 음악 예배에 대한 수요가 폭발적으로 증가하고 있다.
연구에 따르면, 밀레니얼 세대와 Z세대는 음악을 통해 표현되는 예배의 진정성과 신성함을 중요하게 여기며,
현대 팝 록 스타일의 찬양은 청년 문화와 종교를 자연스럽게 연결하는 매개체 역할을 한다(Kang 2014; Marti 2017).

이러한 변화로 인해:
- **찬양 콘텐츠 공급 폭발**: 유튜브, 스트리밍 플랫폼을 통한 고품질 찬양 음원/영상 급증
- **기대 수준 상승**: 전문 워십팀의 음악적 역량에 대한 기대 증가
- **준비 부담 증가**: 더 다양한 곡을 더 높은 수준으로 준비해야 하는 상황

그 결과 악보 관리, 콘티 제작, 팀 협업의 효율성이 그 어느 때보다 중요한 과제로 부상하였다.

### 악보 저작권의 사각지대

그러나 이러한 문화적 성장과는 달리,
찬양 사역을 구성하는 핵심 요소 중 하나인
'악보'에 대한 저작권 인식과 제도는
거의 발전하지 못한 상태에 머물러 있다.

### 현재 문제점

| 문제 | 현재 상황 | 영향 |
|------|----------|------|
| **악보 관리** | 카카오톡, 드라이브, USB에 분산 저장 | 필요할 때 악보를 찾지 못함 |
| **콘티 작성** | 수기 또는 엑셀로 작성 | 변경 시 재작성 필요, 공유 어려움 |
| **악보 필기** | 종이에 직접 필기 또는 PDF 앱 사용 | 악보별 필기 관리 불가, 송폼 표시 어려움 |
| **팀 공유** | 카톡으로 파일 전송 | 버전 관리 불가, 파일 만료 |
| **저작권** | 무단 복사, 관행적 공유 | 법적·윤리적 문제 |

### 시장 규모

| 구분 | 수치 |
|------|------|
| 한국 교회 수 | 약 60,000개 |
| 찬양팀 보유 교회 | 약 40,000개 |
| 잠재 사용자 (국내) | 40만 ~ 60만명 |
| 해외 한인교회 | 5,000개+ |
| 잠재 사용자 (해외) | 5만명+ |

WORSHEEP는 바로 이 지점에서 출발한다.

## Ⅱ. 서비스 및 기술 상세

WORSHEEP는 교회 찬양팀의 예배 준비 전 과정을
하나의 디지털 워크플로로 통합하는 플랫폼이다.

### 핵심 기능

#### 1. 통합 악보 라이브러리
- 1,700곡+ CCM 악보 검색
- 시즌/테마/키/템포/BPM 필터링
- AI 자연어 검색 ("부활절에 부르기 좋은 빠른 찬양")
- 좋아요/즐겨찾기 기능

#### 2. 콘티(예배 순서) 제작
- 드래그 앤 드롭으로 순서 편집
- 고정곡 기능 (여는찬양, 마치는찬양)
- 과거 콘티 복사 및 재사용
- 팀원과 실시간 공유

#### 3. 악보 필기 및 편집
- 펜, 형광펜, 지우개, 텍스트 도구
- 올가미 선택 (다중 객체 편집)
- 송폼 오버레이 (V-C-B-Out)
- 파트 태그 (I, V, C, B 등)
- 피아노/드럼 악보 편집
- Undo/Redo
- Apple Pencil 지원

#### 4. 팀 협업
- 팀 생성 및 초대 코드
- 역할 관리 (Leader, Admin, Member)
- 가입 승인 시스템
- 팀 악보 라이브러리
- 권한 기반 접근 제어

#### 5. 다중 기기 동기화
- 로컬 + 클라우드 하이브리드
- 오프라인 작업 가능
- 자동 동기화

#### 6. 다운로드 및 내보내기
- PDF (송폼 + 파트 태그 포함)
- 이미지 (PNG, JPG)
- 표지 자동 생성
- 콘티 일괄 다운로드

## 콘텐츠 및 저작권 모델

### 악보 콘텐츠 분류

| 분류 | 설명 | 저작권 처리 |
|------|------|-------------|
| **공식 악보** | 저작권자(팀/출판사)와 파트너십 체결 | 사용 허락 계약, 수익 분배 정산 |
| **퍼블릭 도메인** | 저작권 만료된 곡 (저작자 사후 70년) | 자유 이용 가능 |
| **사용자 업로드** | 사용자가 직접 등록한 악보 | 업로더 책임, 공개 범위 선택 가능 |

### 베타 vs 정식 서비스

| 구분 | 베타 (현재) | 정식 서비스 |
|------|-------------|-------------|
| **공식 악보** | 테스트용으로 인기곡 1,700곡+ 등록 | 사용 허락 받은 곡만 유지 |
| **퍼블릭 도메인** | 동일 | 동일 |
| **사용자 업로드** | 동일 | 동일 |

> **중요**: 정식 서비스 출시 시, 파트너십이 체결되지 않은 곡은 삭제되며 공식 악보와 퍼블릭 도메인 곡만 남습니다.

### 사용자 업로드 시스템

#### 업로드 공개 범위

| 범위 | 설명 | 용도 |
|------|------|------|
| **비공개** | 본인만 열람 가능 | 개인 악보 보관 |
| **팀 공개** | 소속 팀원만 열람 가능 | 팀에서 자주 쓰는 곡, 자체 편곡 |
| **전체 공개** | 모든 사용자 열람 가능 | 공유하고 싶은 악보 |

#### 저작권 책임

- 업로드 시 저작권 정책 동의 필수
- **저작권 책임은 업로더에게 있음** (회원가입/업로드 시 동의)
- 전체 공개 악보: 저작권 확인 불가하므로 신고 시 삭제

### 저작권 보호 정책 (Safe Harbor)

저작권법 제102조(온라인서비스제공자의 책임 제한)에 따라 다음 조건 충족 시 플랫폼 면책:

| 조건 | WORSHEEP 대응 |
|------|---------------|
| **저작권 침해 인지 시 즉시 조치** | 신고 접수 후 3영업일 내 삭제 |
| **신고-삭제 절차 마련** | 저작권 정책 페이지에 신고 절차 명시 |
| **반복 침해자 제재** | 3진 아웃제 (경고 → 30일 제한 → 영구 정지) |

#### 신고 처리 절차

```
신고 접수 (이메일) → 검토 (3영업일) → 콘텐츠 삭제 → 업로더 통보 → 이의신청 처리
```

## 결론

WORSHEEP는 단순한 악보 저장 서비스가 아니라,
**예배 준비 → 실행 → 데이터 축적 → 수익 분배**로 이어지는
선순환 구조를 기술적으로 구현한 플랫폼이다.

이 구조를 통해:
- **교회**는 합법적으로 악보를 사용하고 팀 협업이 가능해진다
- **사역팀**은 악보 사용에 대한 정당한 보상을 받는다
- **채보자**는 창작 활동에 대한 수익을 얻는다
- **플랫폼**은 지속 가능한 운영이 가능해진다

WORSHEEP는 교회 찬양 사역이 관행이 아닌,
정당한 구조 위에서 운영되도록 하는 첫 번째 플랫폼이 될 것이다.
