- **기능 검증**: 핵심 기능들의 실제 환경 작동 확인

## 4. 기술 인프라 구현

### 4.1 Google Analytics 4

#### 설치
```bash
npm install @next/third-parties
```

#### 설정 (`app/layout.tsx`)
```tsx
import { GoogleAnalytics } from '@next/third-parties/google'

export default function RootLayout({ children }) {
  return (
    <html>
      <body>{children}</body>
      <GoogleAnalytics gaId="G-XXXXXXXXXX" />
    </html>
  )
}
```

#### 추적할 이벤트
| 이벤트명 | 설명 | 파라미터 |
|---------|------|----------|
| `sign_up` | 회원가입 | method |
| `login` | 로그인 | method |
| `song_view` | 악보 조회 | song_id, song_name |
| `song_download` | 악보 다운로드 | song_id, format |
| `setlist_create` | 콘티 생성 | song_count |
| `team_create` | 팀 생성 | - |
| `annotation_save` | 필기 저장 | song_id |
| `feedback_submit` | 피드백 제출 | type |

### 4.2 Sentry (에러 트래킹)

#### 설치
```bash
npx @sentry/wizard@latest -i nextjs
```

#### 설정
```typescript
// sentry.client.config.ts
import * as Sentry from "@sentry/nextjs";

Sentry.init({
  dsn: process.env.NEXT_PUBLIC_SENTRY_DSN,
  tracesSampleRate: 0.1,
  environment: process.env.NODE_ENV,
});
```

### 4.3 인앱 피드백 버튼

#### DB 테이블
```sql
CREATE TABLE feedbacks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  type TEXT CHECK (type IN ('bug', 'feature', 'other')),
  content TEXT NOT NULL,
  page_url TEXT,
  device_info JSONB,
  status TEXT DEFAULT 'new',
  created_at TIMESTAMP DEFAULT NOW()
);
```

#### 컴포넌트 위치
- `src/components/FeedbackButton.tsx` 생성
- `app/layout.tsx`에서 전역으로 렌더링
- 우측 하단 플로팅 버튼

| 1 | 인앱 피드백 | 상시 | 버그, 건의사항 수집 |

| 3 | 사용자 인터뷰 | 4월 | 5~10명 심층 인터뷰 |

### 7.2 설문조사 (Tally)

#### 중간 설문 (3월) - 주요 질문
1. 전반적 만족도 (1-5)
2. 가장 유용한 기능
3. 가장 개선 필요한 기능
4. 추가 희망 기능
5. NPS (0-10)
6. 유료화 의향

#### 최종 설문 (5월) - 주요 질문
1. 베타 기간 만족도
2. 개선된 점 평가
3. 정식 출시 시 사용 의향
4. 적정 월 구독료
5. 추천 의향

### 7.3 사용자 인터뷰 (4월)

**대상**: 주 3회 이상 접속, 콘티 2개 이상 생성

**질문 가이드**:
1. 주 사용 기능과 불편한 점
2. 이전 방식(카톡, 드라이브)과 비교
3. "없으면 안 될" 정도로 필요한지
4. 유료 구독 의향 및 적정 가격
