## 1. 개요

### 1.1 베타테스트 목적
- **기능 검증**: 핵심 기능들이 실제 환경에서 올바르게 작동하는지 확인
- **사용성 테스트**: UX/UI가 직관적이고 사용하기 쉬운지 검증
- **버그 발견**: 다양한 디바이스/브라우저에서의 버그 식별
- **PMF(Product-Market Fit) 검증**: 실제 찬양팀이 필요로 하는 기능인지 확인
- **저작권 파트너십 확보**: 워십팀과의 공식 파트너십 체결
- **투자 준비**: 사업계획서, VC 투자계획서 작성을 위한 데이터 수집

### 1.2 타겟 사용자
| 구분 | 설명 |
|------|------|
| 1차 타겟 | 교회 찬양팀 리더 (인도자, 밴드 리더) |
| 2차 타겟 | 찬양팀 팀원 (보컬, 악기 연주자) |
| 사용 환경 | 아이패드 (주), 모바일, 데스크탑 |

### 1.3 규모 및 기간
| 항목 | 내용 |
|------|------|
| 베타테스터 목표 | 50~100명 |
| 파트너십 워십팀 목표 | 5~10팀 |
| 테스트 기간 | 2025년 1월 ~ 4월 (약 4개월) |
| 주간 투입 시간 | 25시간 |

### 1.4 핵심 리소스
| 항목 | 현황 |
|------|------|
| 인력 | 1명 (본인) |
| 예산 | Claude 구독, Supabase Pro |
| 본인 교회 찬양팀 | 13명 |
| SNS | 인스타그램 (개설 예정) |

```sql
-- 현재 가사 데이터 현황 확인

  COUNT(*) as total_songs,

  COUNT(*) - COUNT(lyrics) as without_lyrics

```

### 4.3 가사 입력 전략

#### 우선순위 기준
1. **사용 빈도 높은 곡** - 콘티에 자주 사용되는 곡
2. **인기 워십팀 곡** - 마커스, 어노인팅, 제이어스 등
3. **찬송가** - 기본 레퍼토리
4. **시즌 곡** - 크리스마스, 부활절 등

#### 입력 방법
1. **수동 입력** - 앱 내 가사 입력 기능 사용
2. **대량 입력** - Supabase에서 직접 SQL 업데이트
3. **외부 소스 활용** - 공개된 가사 데이터 (저작권 확인 필수)

#### 가사 입력 가이드라인
```

- 한 줄에 한 프레이즈
- 코러스/절 구분 표시 (예: [Verse 1], [Chorus])
- 영어 곡은 영어로, 한글 곡은 한글로
- 반복 가사도 모두 입력

[Verse 1]
```

### 6.2 접근 제한: 가입 승인제

#### 구현 방법

**1. 데이터베이스 스키마**
```sql
-- users 테이블에 status 컬럼 추가
ALTER TABLE users ADD COLUMN status TEXT DEFAULT 'pending';
-- status: 'pending', 'approved', 'rejected'

-- 승인 요청 정보 저장
ALTER TABLE users ADD COLUMN church_name TEXT;
ALTER TABLE users ADD COLUMN role_in_team TEXT;
ALTER TABLE users ADD COLUMN referral_source TEXT;
```

**2. 가입 플로우**
```
1. 회원가입 폼
   - 이메일, 비밀번호
   - 이름
   - 교회명 (필수)
   - 찬양팀 역할 (보컬/건반/기타/드럼/베이스/기타)
   - 가입 경로 (인스타그램/지인소개/기타)

2. 가입 완료 후
   - "가입 신청이 완료되었습니다.
      관리자 승인 후 서비스 이용이 가능합니다.
      승인 완료 시 이메일로 안내드리겠습니다."

3. 승인 대기 상태
   - 로그인 시도 시: "승인 대기 중입니다" 메시지
```

### 7.3 인앱 피드백 시스템

#### 피드백 테이블
```sql
CREATE TABLE feedbacks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  type TEXT CHECK (type IN ('bug', 'feature', 'other')),
  content TEXT NOT NULL,
  page_url TEXT,
  device_info JSONB,
  screenshot_url TEXT,
  status TEXT DEFAULT 'new',

```

#### 피드백 버튼 컴포넌트
```tsx
// components/FeedbackButton.tsx
'use client'
import { useState } from 'react'
import { MessageCircle } from 'lucide-react'

export function FeedbackButton() {
  const [isOpen, setIsOpen] = useState(false)
  const [type, setType] = useState<'bug' | 'feature' | 'other'>('bug')
  const [content, setContent] = useState('')

  const submit = async () => {
    await fetch('/api/feedback', {
      method: 'POST',
      body: JSON.stringify({
        type,
        content,
        pageUrl: window.location.href,
        deviceInfo: {
          userAgent: navigator.userAgent,
          screenWidth: window.innerWidth,
          screenHeight: window.innerHeight,
        }
      })
    })
    setIsOpen(false)
    setContent('')
    alert('피드백이 전송되었습니다!')
  }

  return (
    <>
      <button
        onClick={() => setIsOpen(true)}
        className="fixed bottom-4 right-4 w-12 h-12 bg-purple-600 text-white rounded-full shadow-lg flex items-center justify-center"
      >
        <MessageCircle size={24} />
      </button>

      {isOpen && (
        <div className="fixed inset-0 bg-black/50 flex items-center justify-center p-4 z-50">
          <div className="bg-white rounded-xl p-6 w-full max-w-md">
            <h3 className="text-lg font-bold mb-4">피드백 보내기</h3>

            <div className="flex gap-2 mb-4">
              {(['bug', 'feature', 'other'] as const).map(t => (
                <button
                  key={t}
                  onClick={() => setType(t)}
                  className={`px-3 py-1 rounded-full text-sm ${
                    type === t ? 'bg-purple-600 text-white' : 'bg-gray-100'
                  }`}
                >
                  {t === 'bug' ? '버그' : t === 'feature' ? '기능 제안' : '기타'}
                </button>
              ))}
            </div>

            <textarea
              value={content}
              onChange={(e) => setContent(e.target.value)}
              placeholder="내용을 입력해주세요..."
              className="w-full h-32 p-3 border rounded-lg mb-4"
            />

            <div className="flex gap-2">
              <button
                onClick={() => setIsOpen(false)}
                className="flex-1 py-2 bg-gray-100 rounded-lg"
              >
                취소
              </button>
              <button
                onClick={submit}
                disabled={!content.trim()}
                className="flex-1 py-2 bg-purple-600 text-white rounded-lg disabled:opacity-50"
              >
                전송
              </button>
            </div>
          </div>
        </div>
      )}
    </>
  )
}
```

## 10. 리스크 관리

### 10.1 저작권 리스크

| 리스크 | 발생 가능성 | 영향도 | 대응 방안 |
|--------|-----------|--------|----------|
| 악보 무단 배포 | 중 | 높음 | 가입 승인제, 다운로드 로깅 |
| 저작권자 항의 | 중 | 높음 | 즉시 삭제, 파트너십 제안 |
| 법적 분쟁 | 낮음 | 높음 | 이용약관 면책, 법률 자문 |

**긴급 대응 프로세스**:
```
1. 저작권 항의 접수
   ↓
2. 24시간 내 해당 악보 비공개
   ↓
3. 저작권자에게 사과 연락
   ↓
4. 상황에 따라:
   - 영구 삭제 또는
   - 파트너십 협의 제안
```
