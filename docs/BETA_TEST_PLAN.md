## 1. 개요

### 1.1 베타테스트 목적

- **기능 검증**: 핵심 기능들의 실제 환경 작동 확인
- **사용성 테스트**: UX/UI 직관성 검증
- **버그 발견**: 다양한 디바이스/브라우저 버그 식별
- **PMF 검증**: 실제 찬양팀 니즈 확인

### 5.2 인앱 피드백 (상시) ✅ 구현완료

#### 수집 방법
- 화면 우측 하단 파란색 버튼 클릭
- 유형 선택 (버그/기능제안/기타)
- 내용 작성 후 전송

#### 자동 수집 정보
- 현재 페이지 URL
- 브라우저/기기 정보 (User Agent)
- 로그인 사용자 정보 (있는 경우)

#### 피드백 모니터링
```sql
-- 최근 피드백 확인
SELECT
  type,
  message,
  page_url,
  created_at
FROM feedbacks
ORDER BY created_at DESC
LIMIT 20;

-- 유형별 통계
SELECT
  type,
  COUNT(*) as count
FROM feedbacks
GROUP BY type;
```

- 피드백 2회 이상 제출자

#### 인터뷰 진행 방식
- **방법**: Zoom 또는 대면 (30분~1시간)
- **인원**: 5~10명
- **보상**: 정식 출시 후 3개월 무료 이용권

#### 인터뷰 질문 가이드

**사용 패턴**
1. WORSHEEP을 주로 언제, 어떻게 사용하시나요?
2. 가장 자주 사용하는 기능은 무엇인가요?
3. 한 번 사용할 때 평균 얼마나 사용하시나요?

**이전 방식 비교**
4. WORSHEEP 전에는 어떻게 악보/콘티를 관리하셨나요?
5. 이전 방식과 비교해서 어떤 점이 좋아졌나요?
6. 아직 이전 방식이 더 나은 부분이 있나요?

**Pain Points**
7. 사용하면서 가장 불편했던 경험은?
8. "이것만 고쳐주면 완벽할 텐데" 싶은 것이 있나요?
9. 버그나 오류를 경험한 적 있나요?

**PMF 검증**
10. WORSHEEP이 없어진다면 어떨 것 같으세요?
11. 다른 찬양팀에게 추천하고 싶으신가요? 이유는?
12. 월 1만원 유료화 시 계속 사용하실 의향이 있나요?

**개선 아이디어**
13. 꼭 추가됐으면 하는 기능이 있나요?
14. UI/디자인 측면에서 개선점이 있다면?
15. 마지막으로 하고 싶은 말씀?
