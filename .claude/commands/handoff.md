컨텍스트 리셋(`/clear`) 전에 HANDOFF.md를 업데이트합니다.

## 실행 단계

### 1. 현재 상태 수집
- `git status`로 변경된 파일 확인
- `git log --oneline -5`로 최근 커밋 확인
- `git diff --cached`로 스테이징된 변경사항 확인

### 2. HANDOFF.md 업데이트

기존 HANDOFF.md를 읽고 다음 섹션들을 업데이트:

#### 섹션 1: 프로젝트 현재 상태
- 개발 단계, 등록 곡 수 등 변경사항 있으면 업데이트

#### 섹션 2: 최근 작업
- **마지막 업데이트 날짜** 변경
- **완료된 작업**: 이번 세션에서 완료한 작업들 (체크박스 형식)
- 이전 세션 작업은 간략히 요약하거나 제거

#### 섹션 3: 다음에 할 일
- 완료된 항목은 제거
- 새로 발견된 할 일 추가
- 우선순위 재정렬

#### 섹션 5: 진행 중인 결정 사항
- 새로운 결정 필요 사항 추가
- 결정된 사항은 제거 또는 완료 표시

### 3. 커밋되지 않은 변경사항 처리

커밋되지 않은 변경사항이 있으면:
1. 사용자에게 알림
2. 커밋할지 물어봄
3. 커밋하면 HANDOFF.md와 함께 커밋

### 4. HANDOFF.md 커밋

```bash
git add HANDOFF.md
git commit -m "docs: HANDOFF.md 업데이트 (세션 종료)"
git push
```

### 5. 완료 메시지

```
✅ HANDOFF.md 업데이트 완료

📝 이번 세션 작업:
- [완료된 작업 목록]

📌 다음 할 일:
- [다음 작업 목록 상위 3개]

이제 /clear 해도 됩니다.
새 세션에서 "HANDOFF.md 읽어줘"로 이어서 작업할 수 있습니다.
```

## 작성 원칙

- 간결하게: 핵심 정보만 포함
- 구체적으로: "작업함" 대신 "AI 검색 UI 개선"
- 액션 가능하게: 다음 할 일은 바로 실행 가능한 형태로
