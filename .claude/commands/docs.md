WORSHEEP 프로젝트의 문서를 관리합니다.

## 사용법

```
/docs                    # 문서 목록 확인
/docs list               # 문서 목록 확인
/docs new <문서유형>     # 새 문서 생성
/docs update <문서명>    # 기존 문서 업데이트
```

## 문서 유형

| 유형 | 설명 | 위치 |
|------|------|------|
| `사업계획서` | 내부용 사업 계획 | docs/사업계획서_WORSHEEP.md |
| `협업제안서-팀` | 찬양팀 대상 파트너십 제안 | docs/협업제안서_WORSHEEP.md |
| `협업제안서-채보자` | 채보자 대상 파트너십 제안 | docs/협업제안서_채보자.md |
| `동의서` | 악보 사용 허락 동의서 양식 | docs/동의서_악보사용허락.md |
| `베타가이드` | 베타 테스터 안내서 | docs/베타테스터_가이드.md |
| `서비스소개서` | 1페이지 마케팅 자료 | docs/서비스소개서_WORSHEEP.md |
| `내부운영` | 내부 운영 가이드 | docs/내부운영_가이드.md |
| `FAQ` | 자주 묻는 질문 | docs/FAQ.md |
| `초대메시지` | 베타 테스터 초대 메시지 템플릿 | docs/베타테스트_초대메시지.md |

## 실행 단계

### 목록 확인 시
1. `docs/` 폴더의 모든 마크다운 파일 검색
2. 문서별 제목과 설명 표시

### 새 문서 생성 시
1. 문서 유형에 맞는 템플릿 확인
2. 필요한 정보 수집
3. 문서 생성

### 문서 업데이트 시
1. 기존 문서 읽기
2. 변경할 내용 확인
3. 문서 수정
4. 변경 사항 요약

## 문서 작성 원칙

- 마크다운 형식 사용
- 표, 목록 활용하여 가독성 높이기
- 최신 정보 반영 (날짜, 가격, 기능 등)
- 대상에 맞는 톤 유지 (외부용은 공손하게)
