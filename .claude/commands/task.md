복잡하거나 오래 걸리는 작업의 계획과 진행 상황을 관리합니다.

## 사용법

```
/task              # 현재 진행 중인 작업 확인
/task status       # 현재 진행 중인 작업 확인
/task save         # 현재 작업 상태 저장
/task resume       # 저장된 작업 이어서 진행
/task done         # 작업 완료 처리 (파일 삭제)
```

## 자동 계획 생성 규칙

**다음 조건에 해당하면 작업 시작 전 자동으로 계획 문서를 생성합니다:**

1. **여러 파일 수정이 필요한 작업** (3개 이상)
2. **새 기능 구현** (여러 단계 필요)
3. **리팩토링 작업**
4. **문서 여러 개 수정/생성**
5. **사용자가 "복잡한", "큰 작업", "오래 걸릴" 등 언급 시**

## 계획 문서 형식

파일 위치: `docs/작업계획_현재작업.md`

```markdown
# 작업 계획: [작업명]

**생성일**: 2026-01-19
**상태**: 진행중 / 완료 / 중단

## 1. 작업 개요
[무엇을 하는 작업인지 간단히]

## 2. 작업 항목
- [ ] 항목 1
- [ ] 항목 2
- [ ] 항목 3

## 3. 관련 파일
- `path/to/file1.tsx`
- `path/to/file2.ts`

## 4. 진행 상황
### 2026-01-19
- 항목 1 완료
- 항목 2 진행 중

## 5. 메모
- 주의사항이나 참고할 내용
```

## 실행 단계

### /task 또는 /task status
1. `docs/작업계획_*.md` 파일 검색
2. 있으면 현재 진행 상황 표시
3. 없으면 "진행 중인 작업 없음" 표시

### /task save
1. 현재 작업 상태를 계획 문서에 업데이트
2. 완료된 항목 체크, 진행 상황 기록
3. 다음 단계 명시

### /task resume
1. `docs/작업계획_현재작업.md` 읽기
2. 진행 상황 파악
3. 남은 항목부터 이어서 작업 시작

### /task done
1. 모든 항목 완료 확인
2. 상태를 "완료"로 변경
3. 파일을 `docs/작업완료/` 폴더로 이동 (또는 삭제)

## 주의사항

- 계획 문서는 **항상 최신 상태로 유지**
- 작업 중간에 컨텍스트가 초기화되어도 문서를 읽으면 이어갈 수 있음
- 새 대화에서 `/task resume` 하면 바로 작업 재개 가능
